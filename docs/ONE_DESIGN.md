# Один дизайн — один файл

## Отчёты (прибыли и убытки)

- **Единственная страница отчётов:** `admin-frontend/src/pages/ReportsPage.tsx`
- **Единственный маршрут:** `/reports` → `ReportsPage`
- Блок «Отчет о прибылях и убытках» рендерится только в `ReportsPage.tsx` функцией `renderProfitLossReport()` (компактные стили: `text-base`, `text-xs`, `py-2.5`, `px-4` и т.д.).

**Если не видно изменений в отчёте:**
1. Соберите фронт заново: `npm run build`
2. Задеплойте билд в `/var/www/assetmanagement.team`
3. В браузере: Ctrl+F5 (жёсткое обновление без кэша) или «Очистить кэш и жёсткая перезагрузка»
4. Убедитесь, что открыта вкладка «Прибыли и убытки» (первая в табах отчётов)

Других страниц или компонентов для отчёта о прибылях и убытках в проекте нет.

## Устранение дублей страниц

Ранее в `pages/redesign/` лежали неиспользуемые страницы:
- `DashboardPageRedesign.tsx`
- `AccrualsPageRedesign.tsx`
- `PaymentsPageRedesign.tsx`

В `App.tsx` подключены только `DashboardPage`, `AccrualsPage`, `PaymentsPage` из корня `pages/`. Варианты из `redesign/` в роутер не входили — это были дубли дизайна. Они удалены, чтобы везде был **один файл — один вариант**.

Компоненты в `components/redesign/` (KPIStatCard, Amount, DataTable и т.д.) оставлены: их можно использовать по мере необходимости в основных страницах.
