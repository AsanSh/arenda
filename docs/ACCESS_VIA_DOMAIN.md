# Вход через assetmanagement.team (без порта)

Чтобы заходить в систему **по домену**, а не по `localhost:3000`:

## На сервере (продакшен)

1. **Задеплойте приложение** (на сервере с доменом assetmanagement.team):
   ```bash
   cd /root/arenda/infra
   sudo ./deploy.sh
   ```

2. **Открывайте в браузере:**
   - **https://assetmanagement.team** (если настроен SSL)
   - **http://assetmanagement.team** (если используется nginx без SSL — конфиг `nginx.conf.no-ssl`)

3. **Порт указывать не нужно** — nginx отдаёт фронтенд на порту 80 (или 443 для HTTPS).  
   `localhost:3000` используется только при локальной разработке.

## Локальная разработка

- Запуск фронта: `npm start` в `admin-frontend` — приложение откроется на **http://localhost:3000**.
- Чтобы тестировать «как на домене», после деплоя открывайте **http://assetmanagement.team** (или https) с того же сервера или с машины, где прописан этот домен.

## Итог

| Где        | Адрес входа                |
|-----------|----------------------------|
| Продакшен | **http://assetmanagement.team** или **https://assetmanagement.team** |
| Локально  | http://localhost:3000      |

После входа корень сайта (`/`) перенаправляет на `/dashboard` или на `/login`, если вы не авторизованы.
