# Настройка MCP браузера для входа в приложение через Cursor

Чтобы агент мог открыть Chrome, перейти на страницу логина и войти под пользователем (nimdaSan / nimdaParol) через MCP, нужен **включённый MCP‑сервер браузера**.

## Вариант 1: Встроенный Cursor Browser (cursor-ide-browser)

1. Откройте **Cursor Settings** (Ctrl+, или Cmd+,).
2. Перейдите в раздел **Tools** → **MCP** (или **Features** → **MCP**).
3. Убедитесь, что включён сервер **cursor-ide-browser** или **Cursor Browser** / **Browser Automation**.
4. Если сервер есть в списке, но инструменты не вызываются:
   - Нажмите **Refresh** (обновить) рядом с сервером.
   - Перезапустите Cursor.
5. Для работы нужна **открытая вкладка браузера**, управляемая Cursor (не просто Chrome снаружи). Обычно Cursor сам открывает встроенный браузер при первом использовании MCP.

**Нужные инструменты для входа на логин:**

- `browser_navigate` — переход на URL (например, `http://localhost:3000/login`).
- `browser_snapshot` — снимок страницы и ссылки на элементы.
- `browser_fill` — ввод логина и пароля в поля.
- `browser_click` — нажатие кнопки «Войти».
- `browser_tabs` — список вкладок (при необходимости).
- `browser_lock` / `browser_unlock` — блокировка вкладки для взаимодействия (по инструкции сервера).

Порядок: **navigate → lock → snapshot → fill (логин) → fill (пароль) → click (Войти) → unlock**.

---

## Вариант 2: Browser MCP через mcp.json (npx @browsermcp/mcp)

Если встроенный браузер недоступен, можно добавить отдельный MCP‑сервер браузера через конфиг.

1. Убедитесь, что установлены **Node.js 18+** и **npm 8+**:
   ```bash
   node -v   # v18+
   npm -v    # 8+
   ```

2. Создайте или отредактируйте файл конфигурации MCP:
   - **Linux/macOS:** `~/.cursor/mcp.json`
   - **Windows:** `%USERPROFILE%\.cursor\mcp.json`

3. Добавьте сервер `browsermcp`:
   ```json
   {
     "mcpServers": {
       "browsermcp": {
         "command": "npx",
         "args": ["-y", "@browsermcp/mcp@latest"]
       }
     }
   }
   ```

4. Сохраните файл и в Cursor: **Settings → MCP** нажмите **Refresh** у сервера (или перезапустите Cursor).

5. После этого у агента должны появиться инструменты браузера (navigate, snapshot, click, fill и т.д.).

---

## Проверка

После включения одного из вариантов попросите агента:

- «Открой через MCP браузер страницу логина и войди под nimdaSan / nimdaParol».

Агент должен вызвать `browser_navigate` на `http://localhost:3000/login` (или ваш URL), затем ввести логин/пароль и нажать «Войти».

---

## Если инструменты по-прежнему недоступны

- В Cursor есть два типа MCP: от расширений (например, cursor-ide-browser) и из `mcp.json`. Иногда они не пересекаются — тогда пробуйте второй вариант.
- Убедитесь, что в настройках Cursor в разделе MCP выбран нужный проект/workspace и сервер не отключён для этого проекта.
- Проверьте логи Cursor (Help → Toggle Developer Tools → Console) на ошибки при загрузке MCP.
