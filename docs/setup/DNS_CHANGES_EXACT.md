# Точные изменения DNS записей в Spaceship.com

## 📝 Что нужно изменить

### 1. A запись для основного домена (@)

**Текущее значение:**
```
Host: @
Type: A
Value: 216.24.57.1
TTL: 30 min
```

**Изменить на:**
```
Host: @
Type: A
Value: 5.8.10.197  ← ИЗМЕНИТЬ ЭТО
TTL: 30 min (можно оставить или изменить на 1 hour)
```

**Как изменить:**
1. Найдите строку с `Host: @` и `Type: A`
2. Нажмите на кнопку редактирования (иконка карандаша или "Edit")
3. В поле "Value" замените `216.24.57.1` на `5.8.10.197`
4. Сохраните изменения

---

### 2. CNAME запись для www поддомена

**Текущее значение:**
```
Host: www
Type: CNAME
Value: base44.onrender.com
TTL: 30 min
```

**Изменить на (вариант 1 - заменить на A запись):**
```
Host: www
Type: A  ← ИЗМЕНИТЬ ТИП С CNAME НА A
Value: 5.8.10.197  ← ИЗМЕНИТЬ ЭТО
TTL: 30 min
```

**Или (вариант 2 - оставить CNAME, но изменить значение):**
```
Host: www
Type: CNAME
Value: assetmanagement.team  ← ИЗМЕНИТЬ НА ОСНОВНОЙ ДОМЕН
TTL: 30 min
```

**Рекомендация:** Используйте **вариант 1** (A запись), так как это проще и надежнее.

**Как изменить:**
1. Найдите строку с `Host: www` и `Type: CNAME`
2. Нажмите "Edit" или "Удалить" (если нужно заменить тип)
3. Если удаляете CNAME:
   - Удалите старую запись
   - Создайте новую запись:
     - Host: `www`
     - Type: `A`
     - Value: `5.8.10.197`
     - TTL: `30 min` или `1 hour`
4. Сохраните изменения

---

## ✅ Что НЕ нужно менять

**Оставьте без изменений:**
- ✅ `brevo1._do` (CNAME) - для email
- ✅ `brevo2._dc` (CNAME) - для email
- ✅ `_dmarc` (TXT) - для email
- ✅ `@` (TXT) с `brevo-code:...` - для email

Эти записи нужны для работы email через Brevo и не мешают работе сайта.

## 📋 Итоговая таблица DNS записей

После изменений у вас должно быть:

| Host | Type | Value | TTL | Назначение |
|------|------|-------|-----|------------|
| `@` | A | `5.8.10.197` | 30 min | Основной домен → ваш сервер |
| `www` | A | `5.8.10.197` | 30 min | www поддомен → ваш сервер |
| `brevo1._do` | CNAME | `b1.assetmanagement-team.dkim.brevo.com` | 5 min | Email (не трогать) |
| `brevo2._dc` | CNAME | `b2.assetmanagement-team.dkim.brevo.com` | 5 min | Email (не трогать) |
| `_dmarc` | TXT | `v=DMARC1; p=none; rua=...` | 5 min | Email (не трогать) |
| `@` | TXT | `brevo-code:97436868...` | 5 min | Email (не трогать) |

## ⏱️ После изменений

1. **Сохраните изменения** в панели Spaceship
2. **Подождите 5-30 минут** для распространения DNS
3. **Проверьте изменения:**
   ```bash
   dig assetmanagement.team +short
   # Должен вернуть: 5.8.10.197
   
   dig www.assetmanagement.team +short
   # Должен вернуть: 5.8.10.197
   ```

---

## 🎯 Краткая инструкция

1. **Найдите запись с `Host: @` и `Type: A`**
   - Измените `Value` с `216.24.57.1` на `5.8.10.197`

2. **Найдите запись с `Host: www` и `Type: CNAME`**
   - Удалите эту запись
   - Создайте новую запись:
     - Host: `www`
     - Type: `A`
     - Value: `5.8.10.197`

3. **Сохраните все изменения**

4. **Подождите 5-30 минут**

5. **Проверьте:** `dig assetmanagement.team +short`

---

## ⚠️ Важно

- Не удаляйте записи для Brevo (email сервис)
- После изменений DNS может потребоваться до 48 часов для полного распространения (обычно 5-30 минут)
- Убедитесь, что на сервере уже установлен и настроен nginx (см. `DOMAIN_SETUP_STEPS.md`)
