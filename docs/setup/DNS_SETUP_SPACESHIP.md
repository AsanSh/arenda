# Настройка DNS записей в Spaceship.com для assetmanagement.team

## Пошаговая инструкция

### 1. Вход в личный кабинет Spaceship.com

1. Откройте https://spaceship.com/
2. Войдите в свой аккаунт
3. Перейдите в раздел "Domains" или "Мои домены"
4. Найдите домен `assetmanagement.team`

### 2. Настройка DNS записей

#### Вариант 1: Через панель управления Spaceship

1. Нажмите на домен `assetmanagement.team`
2. Найдите раздел "DNS Management", "DNS Records" или "Управление DNS"
3. Удалите все существующие A записи для `@` и `www` (если есть)
4. Добавьте новые записи:

**A запись для основного домена:**
```
Тип записи: A
Имя/Хост: @ (или оставьте пустым, или assetmanagement.team)
Значение/IP: 5.101.67.195
TTL: 3600 (или Auto)
Приоритет: (не требуется для A записи)
```

**A запись для www поддомена:**
```
Тип записи: A
Имя/Хост: www
Значение/IP: 5.101.67.195
TTL: 3600 (или Auto)
Приоритет: (не требуется для A записи)
```

#### Вариант 2: Если Spaceship использует внешний DNS провайдер

Если Spaceship предоставляет только домен, но DNS управляется через другого провайдера (например, Cloudflare, Namecheap и т.д.):

1. Найдите в настройках домена информацию о DNS серверах (Nameservers)
2. Если DNS управляется через другого провайдера, настройте записи там
3. Используйте те же A записи, что указаны выше

### 3. Проверка DNS записей

После настройки подождите 5-15 минут и проверьте:

```bash
# Проверка A записи для основного домена
dig assetmanagement.team +short
# Ожидаемый результат: 5.101.67.195

# Проверка A записи для www
dig www.assetmanagement.team +short
# Ожидаемый результат: 5.101.67.195

# Проверка через Google DNS
dig @8.8.8.8 assetmanagement.team +short

# Проверка через Cloudflare DNS
dig @1.1.1.1 assetmanagement.team +short
```

### 4. Время распространения DNS

- **Минимальное время:** 5-15 минут
- **Обычное время:** 1-2 часа
- **Максимальное время:** до 48 часов (редко)

### 5. Дополнительные записи (опционально)

Если нужно настроить поддомены:

**API поддомен (если планируется отдельный поддомен для API):**
```
Тип: A
Имя: api
Значение: 5.101.67.195
TTL: 3600
```

**CNAME для www (альтернатива A записи):**
```
Тип: CNAME
Имя: www
Значение: assetmanagement.team
TTL: 3600
```

### 6. После настройки DNS

1. ✅ Дождитесь распространения DNS (проверьте через `dig`)
2. ✅ Установите nginx на сервере (см. DOMAIN_SETUP.md)
3. ✅ Получите SSL сертификат через certbot
4. ✅ Настройте nginx конфигурацию
5. ✅ Обновите настройки Django и фронтенда

## Troubleshooting

### DNS записи не обновляются

1. Проверьте, что записи сохранены в панели Spaceship
2. Очистите DNS кэш на вашем компьютере:
   ```bash
   # Windows
   ipconfig /flushdns
   
   # Linux/Mac
   sudo systemd-resolve --flush-caches
   # или
   sudo dscacheutil -flushcache
   ```
3. Попробуйте проверить через разные DNS серверы

### Неправильный IP адрес

Убедитесь, что IP адрес сервера правильный:
- Текущий IP: `5.101.67.195`
- Если IP изменился, обновите A записи

### Поддомен не работает

- Убедитесь, что создана отдельная A запись для поддомена
- Или используйте CNAME запись, указывающую на основной домен
