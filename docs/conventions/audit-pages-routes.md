Актуально на момент аудита. После изменений роутов или добавления страниц — обновить этот файл.

## Страницы без маршрута (legacy / дубликаты)

Эти файлы есть в `src/pages/`, но **не подключены** в `App.tsx`:

| Файл | Назначение | Рекомендация |
|------|------------|--------------|
| **LoginPageNew.tsx** | Альтернативная страница входа (WhatsApp QR) | Не используется. Единая точка входа — **LoginPage** (вкладки Логин/Пароль и WhatsApp OTP). Можно удалить или оставить как референс. |
| **LoginPageOTP.tsx** | Вход только по OTP | Функциональность перенесена в **LoginPage** (вкладка WhatsApp). Можно удалить или оставить как референс. |
| **ForecastPage.tsx** | Отдельная страница «Прогноз» | Прогноз перенесён во вкладку внутри **ReportsPage** (таб «Прогноз»). Маршрут `/forecast` редиректит на `/reports?type=forecast`. Страницу можно удалить или оставить для совместимости вызовов API. |

## Конвенции по страницам и роутингу

1. **Вход в систему** — один компонент: **LoginPage**. В нём две вкладки: «Логин/Пароль» и «WhatsApp» (запрос кода → ввод кода). Не добавлять отдельные роуты для альтернативных страниц входа без согласования.
2. **Прогноз** — доступен только через раздел **Отчет** (`/reports`), таб «Прогноз». Отдельного пункта «Прогноз» в боковом меню нет; старый URL `/forecast` редиректит на `/reports?type=forecast`.
3. **Двойные страницы** — не заводить две страницы с одной и той же функциональностью (например, два варианта логина в роутинге). Либо одна страница с вкладками/режимами, либо явный редирект.
4. **Новые страницы** — добавлять в `App.tsx` и при необходимости в боковое меню (`hooks/useUserMenu.ts`). Проверять, что нет дублирования по смыслу с существующими страницами.

## Бэкенд API (кратко)

- Роуты API под префиксом `/api/`: auth, dashboard, properties, contracts, accruals, payments, deposits, account, accounts, forecast, reports, notifications, core (tenants, requests, settings и т.д.).
- Прогноз: API `/api/forecast/` используется и из ReportsPage (таб «Прогноз»), и раньше из ForecastPage — дублирования по API нет, один источник данных.
