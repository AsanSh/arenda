## Защищённые учётные записи администраторов (обязательны)

**Два администратора должны всегда быть доступны** и не могут быть отключены или удалены:

| Логин    | Пароль   | Назначение        |
|----------|----------|--------------------|
| nimdaSan | nimdaParol | Админ для входа по логину/паролю и тестов |
| Bahi     | BPHolding  | Админ владельца (BP Holding)             |

**Правила конвенции:**

- **Держать учётные записи всегда активными:** для nimdaSan и Bahi в БД и в коде всегда должно быть `is_active = True`. Не отключать их, не удалять логику, которая это гарантирует.
- В коде эти логины заданы в `backend/core/models.py` как `PROTECTED_ADMIN_USERNAMES`; при сохранении `User` у них принудительно выставляется `is_active = True`.
- В `backend/core/auth_views.py` (LoginView): при входе под защищённым логином, если учётка по какой-то причине оказалась неактивной — принудительно включаем (`is_active = True`, save) и разрешаем вход.
- В Django Admin для этих пользователей поле «Активен» только для чтения — деактивировать нельзя.
- Создание/обновление учёток: команда `python manage.py create_login_users` (создаёт nimdaSan и Bahi, если ещё нет).
- **Не удалять** эти логины из `PROTECTED_ADMIN_USERNAMES`, не убирать команду `create_login_users` и не убирать принудительное включение учётки при логине — они обеспечивают гарантированный доступ в систему.

**Автоматическое назначение администратора:** при входе по логину **nimdaSan** или **Bahi**, либо по номеру **+996700750606**, пользователь всегда получает роль администратора (`role=admin`, `is_staff=True`, `is_superuser=True`). Это обеспечивается в коде: `ensure_protected_admin()` в `core/models.py`, вызов при логине (auth_views) и в `/auth/me/`, а также в WhatsApp-авторизации. Номер задан в `ADMIN_PHONES` в `core/models.py`. Команда `python manage.py ensure_protected_admins` приводит существующие учётки в БД к роли админа.

## Типы пользователей

### 1. Администратор (administrator)
**Полный доступ ко всему.**
- Может создавать, редактировать, удалять всё
- Видит всю аналитику
- Управляет пользователями и настройками

### 2. Владелец компании (owner)
**Управление бизнесом.**
- Видит всю финансовую информацию
- Может создавать договоры, объекты
- Не может удалять (только администратор)

### 3. Хозяин недвижимости (landlord)
**Владелец объектов.**
- Видит только свои объекты
- Видит договоры и платежи по своим объектам
- Может редактировать информацию о своих объектах

### 4. Инвестор (investor)
**Пассивный инвестор.**
- Видит отчёты по прибыли
- Видит ROI по своим инвестициям
- Не может редактировать

### 5. Арендатор (tenant)
**Квартиросъёмщик.**
- Видит только свои договоры
- Видит свои начисления и платежи
- Может вносить платежи

### 6. Сотрудник (employee)
**Операционная работа.**
- Видит всё (как administrator)
- Может создавать, редактировать
- Не может удалять

### 7. Мастер (master)
**Обслуживание объектов.**
- Видит объекты для обслуживания
- Может добавлять заметки о состоянии
- Не видит финансы

## Как добавить нового пользователя

1. Создать контрагента в разделе «Контрагенты».
2. Указать номер телефона.
3. Выбрать тип пользователя (администратор, владелец, арендатор и т.д.).
4. Пользователь входит по WhatsApp OTP по этому номеру и получает права согласно типу.
